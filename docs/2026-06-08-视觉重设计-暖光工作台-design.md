# 视觉重设计：暖光工作台 + 夜间模式

日期：2026-06-08
范围：`index.html` 视觉层重做。**不改**功能逻辑、`server.js`、`sessions.js`、存档结构。保持单文件、零依赖、不引入外部图片资源。

## 背景与问题

当前页面是一个能跑的工具，但视觉上有几处明显短板（基于 1280×820 实测）：

1. **主舞台约 70% 是死白**，拼豆盘缩在左上角，全屏最小的元素却是唯一的交互主体，缺焦点、像没加载完。
2. **配色发灰、对比不足**：米白底 + 奶白面板 + 米灰线条三层"米"叠在一起，读起来是"褪色/未完成"而非"温暖治愈"。
3. **缺氛围与质感**：全是纯色平涂，没有层次和"豆子"的触感。
4. **顶栏按钮平庸且无主次**：导航类（选图案/作品墙）与工具类（绘制/橡皮/检查）长得一样。
5. **底部信息零散**：缩放、适配、熨烫、进度、调色板堆成几条灰扁横栏。
6. **缺世界观**：没有"等代码间隙、坐在暖工作台前拼一颗豆"的场景情绪。

## 目标

把页面从"能跑的工具"提升为"让人想停下来玩两分钟的解压小物"，建立一套完整的暖光视觉语言，并新增夜间模式。

## 一、设计语言：暖光工作台

核心隐喻——**关了顶灯，留一盏暖台灯，你坐在工作台前拼一颗豆**。白天与夜间是「同一张桌子的两个时刻」，共用同一套结构与隐喻，仅换配色，不是两套割裂皮肤。

关键手法是**远近层次**：台面最深 → 盘面底板浮起 → 豆子/卡片最亮。用层次（投影、明暗、浮起关系）建立对比，而非堆饱和色。这直接根治"三层米叠一起发灰"。

风格基调：**克制的暖光**，非重拟物木纹；质感用细腻 CSS 渐变实现，浓淡可调。

## 二、配色 token（两套主题，全部走 CSS 变量）

所有颜色收敛到 `:root`，新增 `[data-theme="night"]` 覆盖块。canvas 绘制读取当前主题色。

### ☀️ 白天（暖光台面）
- 台面背景 `--desk`：暖木棕 `#e6d8c0`；右上角暖光晕 `#fff4e0`（径向渐变往左下渐暗，模拟台灯打光）
- 盘面底板 `--board-base`：浅硅胶 `#f3ece0`
- 卡片/面板 `--panel`：`#fffdf8`
- 墨色 `--ink` `#4a4036` / 次要 `--muted` `#9a8f80`
- 强调 `--accent` 暖橙 `#ff8a5b` / `--accent2` 琥珀 `#ffb454`
- 分隔线 `--line` `#ece2d3`

### 🌙 夜间（暖夜灯，不是冷科技黑）
- 台面背景 `--desk`：深暖褐 `#221b15`；右上角暖橙台灯光晕
- 盘面底板 `--board-base`：深暖灰棕 `#352c23`
- 卡片/面板 `--panel`：`#2e2620`（细边带微光）
- 墨色 `--ink` 暖米白 `#f0e6d6` / 次要 `--muted` `#a89880`
- 强调 `--accent` 更亮暖橙 `#ff9d6b` / `--accent2` `#ffbf7a`
- 分隔线 `--line` `#473c30`
- 点亮的豆子在暗底上**自带柔光晕**（夜里点灯的治愈感；暖色 glow，不发冷）

状态色（ok/wait/done/running 等）在两套主题下都需重新校准对比度，保证 pill / 脉搏 / 进度可读。

## 三、组件级改动

### 1. 拼豆盘（核心）
- 默认**居中 + 自动适配填满舞台**，彻底消除死白。
- 嵌进圆角 peg board 底板：边缘高光 + 投影，网格为一个个凹下去的桩孔。
- 豆子重绘：环形高光（甜甜圈式塑料反光）+ 底部微投影；空孔画成凹陷 + 淡编号。
- 点亮动效：scale 回弹 + 一圈柔光扩散；夜间额外带 glow。

### 2. 顶栏（分主次）
- 左：logo。
- 导航组（选图案 / 作品墙）与工具组（绘制 · 橡皮 · 检查，做成 segmented 工具组带选中高亮）视觉分离。
- 🔥 熨烫保持"就绪召唤"高光脉冲态。
- 新增 **🌙/☀️ 主题切换**、🔊。
- 顶栏为半透明暖白/暖深条，贴在台面上方，带细底投影。

### 3. 底部操作台（收成一个 dock）
- 上排：图案名 + 缩放滑块 + 适配 + 进度条（暖色质感填充）+ 百分比。
- 调色板做成**"豆子盒"**：一格格凹槽，内嵌立体感实体豆子，显示编号 + 剩余数，选中态明显，拼完打勾变暗。

### 4. 右侧 CC 进度面板
- 卡片统一浮起、投影、圆角；运行中脉搏 / 状态 pill 两套主题下校准对比度；整体随主题换色。
- 项目分组（可折叠）、筛选弹层沿用同一卡片语言。

### 5. 完成反馈与弹窗
- 完成撒花 / banner / toast 适配暖色与夜间。
- 选图案 modal、作品展示墙、作品详情 modal 全部沿用台面 + 卡片语言，适配两套主题。

## 四、主题切换逻辑
- 顶栏按钮切换，挂 `<html data-theme="day|night">`，换变量即换肤，**200–300ms 平滑过渡**。
- **初始默认跟随系统** `prefers-color-scheme`；用户手动切一次后，写入 localStorage 并记住，之后不再跟随系统。
- localStorage key 独立，不影响既有存档结构。

## 五、动效基调
克制柔和（解压系，不花哨）：
- 主题过渡 200–300ms 颜色渐变。
- 豆子点亮：回弹 + 柔光（夜间带 glow）。
- 按钮 hover：轻微浮起。

## 六、技术实现约束
- 颜色全部收敛到 CSS 变量 + `[data-theme="night"]` 覆盖块。
- 背景暖光、底板、桩孔、噪点质感全部用 CSS 渐变实现，**不引入任何图片资源**，保持单文件零依赖。
- canvas 豆子绘制参数（高光/投影/glow）随当前主题读取。
- 主题状态：读系统偏好为默认 → localStorage 覆盖记忆。

## 七、不做（Out of Scope）
- 不改任何功能逻辑、交互流程、数据结构。
- 不动 `server.js` / `sessions.js` / hooks。
- 不改既有 localStorage 存档/作品廊数据格式。
- 不引入构建步骤、依赖或外部资源。

## 八、验收标准
1. 默认进入页面，拼豆盘居中并自适配填满舞台，无大片死白。
2. 白天主题呈现明确的远近层次（台面 < 盘面 < 豆子/卡片），不再"发灰"。
3. 豆子有环形高光 + 投影，点亮有回弹 + 柔光反馈。
4. 顶栏导航组与工具组视觉可区分。
5. 底部为结构化操作台，调色板为"豆子盒"样式。
6. 🌙/☀️ 切换平滑，全部组件（含弹窗/撒花/面板）正确换肤；夜间豆子有暖色 glow。
7. 主题默认跟随系统，手动切后被记住。
8. 功能与既有存档完全不受影响；页面仍为单文件零依赖。
