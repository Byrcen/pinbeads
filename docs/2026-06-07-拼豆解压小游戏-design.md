# 拼豆解压小游戏 — 设计文档（Spec）

- 日期：2026-06-07
- 定位：在等待 Claude Code / Codex 写代码的碎片时间里，用「照图拼豆」替代刷短视频，放松又不沉迷；并与 CC 联动，代码跑完时把注意力拉回终端。
- 形态：本地 `拼豆/` 文件夹（放在桌面），零依赖 Node 桥 + 自包含 HTML 游戏页。

---

## 1. 目标与非目标

### 目标
- 照图拼豆（color-by-number）核心玩法：选色 → 点格子填豆 → 拼完庆祝。
- 内置一批可爱像素图，开箱即玩；同时支持上传图片自动转模板。
- 解压体验：柔和音效、完成点亮动画、进度自动保存随时续玩。
- 与 Claude Code 联动：实时显示各 CC 会话状态/活跃脉搏，完成时四合一提醒（系统通知 / 提示音 / 页内横幅 / 标题闪烁）。多窗口同时显示。
- 防沉迷：连续游玩温和提醒；完成提醒本身作为最强拉回信号。

### 非目标（YAGNI）
- 不做真实「完成百分比」——CC 不提供，改用「状态 + 活跃脉搏」。
- 不做账号 / 云同步 / 多人；存档仅本地 localStorage。
- 不做 WebSocket（用 SSE，零依赖即可）。
- 不做关卡/计时/消除等刺激玩法（刻意避免沉迷）。

---

## 2. 技术形态

- **方案 A（已选）**：本地 `拼豆/` 文件夹，零依赖 Node 桥 `server.js` + 自包含 `index.html`。
- 启动方式：`node server.js` → 浏览器打开 `http://localhost:4500`。
- 一次性设置：往 `~/.claude/settings.json` 加几行 hooks（curl 转发，`|| true` 容错）。
- 选型理由：契合用户「单文件 HTML 成品」习惯；浏览器沙箱无法直接读 CC 状态，必须经本地桥；SSE + curl 全部零依赖、macOS 自带。

---

## 3. 架构与模块

```
拼豆/
  index.html   # 游戏本体（UI / Canvas 渲染 / WebAudio 音效 / CC 面板 / 存档 / 防沉迷，自包含）
  server.js    # 零依赖 Node 桥：serve 页面 + 接收 hooks + SSE 推送
  README.md    # 启动说明 + settings.json hooks 配置片段
  docs/        # 本设计文档
```

模块（同文件内的独立函数/对象，边界清晰、可独立理解）：

- **Pattern（图案模型）**：网格 + 调色板，所有玩法围绕它。
- **PatternLibrary（内置图案库）**：8~10 张手作像素图，紧凑字符串编码。
- **ImageConverter（图片转模板）**：图片 → 降采样 → 量化到拼豆色板 → Pattern。
- **Board（Canvas 渲染）**：底层编号提示 + 上层已放置豆子；缩放/平移。
- **Audio（WebAudio 合成）**：放豆音 / 完成音 / CC 提醒音，全部代码合成无外部文件。
- **Storage（存档）**：localStorage 自动存进度 + 完成作品廊。
- **CCPanel（联动面板）**：EventSource 接 SSE，渲染会话卡片，触发完成提醒。
- **FocusGuard（防沉迷）**：连续游玩计时温和提醒。

## 4. 数据模型

### Pattern
```
Pattern = {
  id: string,            // 内置图固定 id；上传图用内容哈希
  name: string,
  width: number, height: number,
  palette: [{ idx: number, hex: string, name: string }],
  cells: Int8Array,      // 长度 w*h，值=颜色 idx，-1=空白（无豆）
}
```

### 内置图案编码
```
{ name, legend: { 'r': '#e74c3c', ... }, rows: ['..rr..', 'rrrrrr', ...] }   // '.' = 空白
```

### 进度（存档）
```
Progress = {
  patternId: string,
  placed: Uint8Array,    // 0/1 per cell，哪些目标格已填
  updatedAt: number,
}
```
- key：`pinbeads:progress:<patternId>`；另有 `pinbeads:gallery`（完成作品缩略图列表）、`pinbeads:settings`（音量/静音/防沉迷阈值/端口）。

### 拼豆色板（ImageConverter 量化目标）
- 固定 ~28 种常见拼豆色（含中性灰阶 + 高饱和基色），每色 { hex, name }。
- 量化：每个降采样块取平均色 → 取色板中加权 RGB 距离最近者；近白/透明 → 空白(-1)。

## 5. 玩法交互（照图填色）

- **调色板条**：每色一个色块 + 编号 + 「剩余 N 颗」。点选激活当前色。
- **填豆**：点按/拖拽格子 —— 目标色==当前色 → 放豆（弹入动画 + 音效 + 计数-1 + 自动存档）；目标色≠当前色 → 温和忽略（轻微抖动，无惩罚）；空白格不可点。
- **进度反馈**：某色拼完 → 色块打勾变暗；整图拼完 → 扫光 + 撒花 + 完成音。
- **辅助**：可选「高亮当前颜色所有格子」开关，降低找格负担。
- **大图缩放**：缩放滑块 + 一键适配；非绘制时拖拽平移。

### Board 渲染细节
- 空白目标格：淡淡目标色底 + 小号编号（color-by-number 观感）。
- 已放置豆：拼豆风格——圆角方块带中心圆孔（donut），轻微高光做 3D。
- 网格线细淡；单元格尺寸随容器自适应。

---

## 6. 图片转模板流程

1. 文件选择 / 拖拽放入图片。
2. 选网格宽度（16 / 24 / 32 / 48），高度按原图宽高比换算。
3. 绘到离屏 Canvas → 按块降采样取平均色。
4. 量化到拼豆色板（最近色），近白/透明 → 空白。
5. 生成带编号 Pattern，内容哈希作为存档 id。

---

## 7. 音效（WebAudio 合成）

- **放豆音**：~120ms 短音（sine/triangle），带 ±随机音高与快速衰减，「啵/咕」感（快速降调）。
- **完成音**：小段上行琶音 chime。
- **CC 提醒音**：明显区别于拼豆音的双音「叮咚」，略响、易辨识。
- 全局静音开关 + 音量设置；尊重 `pinbeads:settings`。

## 8. CC 联动（核心）

### 数据流
```
CC 窗口 ─(SessionStart/PostToolUse/Notification/Stop/SubagentStop hook)→ curl POST localhost:4500/hook
                                                                              │
                                                                      server.js（零依赖）
                                                                  按 session_id 维护会话表
                                                                              │  SSE
                                                                              ▼
                                                              index.html（EventSource /events）
                                                                  侧栏会话卡片实时刷新
```

### server.js 端点
- `GET /` → serve `index.html`（及内联资源）。
- `POST /hook` → body 为 CC hook 的 stdin JSON（含 `hook_event_name`、`session_id`、`cwd`、可能的 `tool_name`/`message`）；更新会话表并向 SSE 客户端广播。
- `GET /events` → SSE 流；连接时先推全量快照，之后推增量。
- 会话表：`session_id → { id, project(cwd basename), state, toolCalls, lastTool, startedAt, lastEventAt, completedAt }`。
- 清理：空闲 > 30 分钟的会话移除；已完成会话保留并标记，稍后过期。

### 事件 → 状态映射
- `SessionStart` → 新建/重置会话，state=`running`，记 startedAt。
- `PostToolUse` → state=`running`，toolCalls++，记 lastTool / lastEventAt（活跃脉搏）。
- `Notification` → state=`waiting`（等你输入）。
- `Stop` → state=`done`，记 completedAt。
- `SubagentStop` → 活跃脉搏（v1 可仅刷新 lastEventAt）。

### 游戏侧 CC 面板
- 侧栏列出所有会话卡片：项目名 / 状态药丸（🟢 运行中 · 🟡 等你输入 · ✅ 已完成）/ 脉搏点（最近活动、工具调用次数）/ 已耗时计时；按最近活动排序，多窗口同时显示。
- **完成提醒（四合一）**：会话转入 `done` 时同时触发——①系统桌面通知（Web Notifications，需一次授权）②CC 提醒音 ③页内顶部横幅 + 该卡片高亮跳动 ④标签标题闪烁（如「✅ 已完成!」）。
- `waiting` 给较轻的同款提醒（药丸变色 + 可选通知）。

### Hooks 配置（写入 README，供用户粘贴到 ~/.claude/settings.json）
- 每个事件运行 curl 转发 stdin：
  `curl -s -X POST -H 'content-type: application/json' --data-binary @- http://localhost:4500/hook || true`
- 注册事件：SessionStart、PostToolUse（matcher `*`）、Notification、Stop、SubagentStop。

## 9. 防沉迷 + 存档

- **存档**：每放一颗豆自动存 localStorage；重开自动续拼最近图案；完成作品进「我的拼豆」缩略图廊。
- **防沉迷**：连续游玩 ~10 分钟（可配）弹温和 toast「去看看代码？」；CC 完成提醒作为最强主动拉回信号。

---

## 10. 错误处理与边界

- 桥未启动：游戏页正常运行（纯拼豆可玩），CC 面板显示「未连接，运行 node server.js」；hooks 带 `|| true` 不阻塞 CC。
- 端口占用：server.js 可读环境变量/配置改端口；游戏页端口与之对应（README 说明）。
- SSE 断线：EventSource 自动重连；重连后服务端重推全量快照。
- 上传非图片/超大图：校验类型；超大图先压缩再降采样，给尺寸上限提示。
- localStorage 满/不可用：降级为不存档并提示。

## 11. 测试要点

- Pattern/ImageConverter：给定图片与网格宽度，量化输出尺寸与色板映射符合预期；近白→空白。
- 玩法：正确色放豆成功、错误色被忽略、计数与完成判定正确。
- Storage：放豆后刷新页面可续拼；完成入廊。
- 桥：模拟各 hook 事件 POST，会话表状态机正确；SSE 客户端收到全量+增量。
- 提醒：done 事件触发四类提醒（通知权限被拒时其余三类仍工作）。
- 容错：桥关闭时游戏可玩、hooks 不报错；SSE 断线重连。
