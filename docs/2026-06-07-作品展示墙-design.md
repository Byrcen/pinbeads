# 拼豆作品展示墙 — 设计文档

- 日期：2026-06-07
- 定位：把「我的作品」从弹窗网格升级为一个全屏、沉浸式的**洞洞板展示墙**，像现实里拼豆爱好者的展示空间——大件用挂钩挂上墙，小件做成钥匙串吊在挂杆上。

---

## 1. 目标 / 非目标

### 目标
- 全屏独立展示空间（不是弹窗），进入像走进一间作品房，✕/返回退回游戏。
- 洞洞板（pegboard）墙面：暖木色面板 + 规则圆孔点阵，沿用项目暖色 + 点阵美学。
- 按尺寸自动分区：`Math.max(w,h) >= 28` → 大件区（挂钩挂洞洞板）；否则 → 钥匙串区（钥匙环吊挂杆）。
- 点作品 → 详情卡：放大图 + 名字 + 尺寸 + 完成时间，操作：**重新拼这张 / 重命名 / 删除**。

### 非目标（YAGNI）
- 不做拖拽排序 / 自定义摆放位置（自动布局）。
- 不做导出 / 分享 / 实体下单。
- 不保留旧的 galleryModal 弹窗（被展示墙替换）。

---

## 2. 数据改动

作品项扩展：`{id, name, at, thumb}` → `{id, name, at, thumb, w, h}`。
- `addToGallery()` 写入时带上 `state.pattern.width/height`。
- 大小判定：`isBig = Math.max(w,h) >= 28`。
- 老作品缺 `w/h`：渲染时回退解析——内置图查 `PATTERNS`，上传图查 `pinbeads:imgdef:<id>`；都查不到则默认归小件（钥匙串）。

存储 key 不变：`pinbeads:gallery`。

---

## 3. 结构与渲染

- 新增全屏层 `#showcase`（默认隐藏，`.show` 显示），含标题栏 + 洞洞板墙面 `#wall`。
- `buildShowcase()`：读 `pinbeads:gallery`，按 isBig 分到大件区/钥匙串区，分别渲染。
  - 大件：像素缩略图嵌小相框 + 顶部挂钩(CSS/canvas) 扣进洞洞孔，松散网格。
  - 小件：缩略图 + 金属钥匙环 + 短链，从底部挂杆吊成一排。
  - 空状态：「还没有作品，去拼一个挂上来吧～」。
- 洞洞板背景：`radial-gradient` 重复圆孔点阵 + 暖木色。
- `btnGallery` 改为打开 `#showcase`；删除 `galleryModal` 及旧 `buildGallery`。

## 4. 交互（详情卡 `#workDetail`）

点任意作品 → 弹出详情卡（复用 `.modal/.sheet` 风格）：
- 放大豆子图（`renderThumb` 大尺寸）+ 名字 + 尺寸 `W×H` + 完成时间。
- **重新拼这张**：内置图 `PATTERNS.find→decodePattern`，上传图 `imgdef→deserializePattern`，`loadPattern` 后关展示墙。复用现有恢复逻辑。
- **重命名**：`prompt` 改名，写回 gallery，重渲染。
- **删除**：复用现有删除逻辑（按 `id+at` 匹配），重渲染。

## 5. 边界
- gallery 为空：墙面显示空状态提示。
- 「重新拼」时源图案查不到（内置图被删/imgdef 丢失）：toast 提示「找不到原图，无法重拼」，不崩。
- 缩略图缺失：占位灰块。

## 附录：未完成进度保存（2026-06-07 追加）

**问题**：放豆存档有 120ms 节流，若放完豆立即切换图案，节流未触发，内存进度被新图覆盖 → 进度丢失。且上传图未拼完后无处可回（不在选图案列表、不在作品墙）。

**修复**：
- `persistNow()`：同步落盘当前进度（progress + last + imgdef）并更新 WIP 登记表；`saveProgress()` 改为它的节流包装。
- `loadPattern()` 切换前先 `persistNow()` 落盘旧图；`beforeunload` / 标签隐藏时也落盘。
- **WIP 登记表 `pinbeads:wip`**：记录所有「已开拼未拼完」的作品（含上传图：id/name/w/h/placed/total/thumb）。完成入墙（`addToGallery`）时移除。
- **「继续拼（未完成）」列表**：在「选图案」弹窗顶部，展示 WIP（不含当前正在拼的）+ 进度%，点击 `resumeWip` 续拼，✕ `discardWip` 丢弃（删 WIP + progress key）。原图查不到时提示并自动丢弃。
