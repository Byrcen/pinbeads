# 视觉重设计：像素风 UI（泰拉瑞亚 · 木质/羊皮纸）

日期：2026-06-12
范围：`index.html` 的 UI 框架视觉层（CSS 为主战场）+ `server.js` 新增字体静态路由 + 仓库新增 `assets/fonts/` 字体文件。**不改**游戏逻辑、存档结构、CC 联动协议、`sessions.js`；**不改**盘面/洞洞板/豆子的渲染（canvas 绘制代码与其消费的变量值原样保留）。

## 背景与目标

现有「暖光工作台」风格圆润柔和。本次把整个操作界面（chrome：顶栏、按钮、面板、调色板、操作台、弹窗、作品廊、toast 等）改成**泰拉瑞亚式像素风**，方向定为**木质/羊皮纸**——木屋工作台感：木框面板、羊皮纸底、金色标题。核心隐喻顺势升级：**拼豆板摆在一张像素木桌上**——界面是像素游戏，作品是真拼豆，两层质感形成有意的对照。

## 已确认的决策（与用户逐条确认）

1. **范围**：只改 UI 框架；盘面、洞洞板、豆子保持现有拟真质感（夜间豆子辉光照旧）。
2. **主题关系**：替换式改造，不留旧皮肤开关；日/夜切换保留——日间「亮堂木屋」、夜间「油灯木屋」，🌙 按钮照常用。
3. **字体**：捆绑开源中文像素字体「缝合怪 Fusion Pixel 12px proportional 简中子集」（OFL-1.1，woff2 约 1-2MB）进仓库，全站中英数统一像素字。
4. **配色基调**：木质/羊皮纸（非经典库存蓝、非纯保留暖色）。
5. **技术路线**：方案 A「原地改造」——保留 CSS 变量架构（`:root` / `[data-theme="night"]`），替换变量值 + 原地改写圆角/阴影/边框等结构性手法；不做 `data-skin` 双皮肤覆盖层，回滚靠 git。

## 一、视觉语言

### 配色 token——☀️ 日间「亮堂木屋」
- 台面 `--desk`：暖木棕 ≈`#b8854f`，叠 `repeating-linear-gradient` 细暗线做木板拼缝
- 面板 `--panel`：羊皮纸米黄 ≈`#f4e4c1`；内层卡片 `--panel-2` ≈`#faf0d8`
- 墨色 `--ink` ≈`#4a3220`；次要 `--muted` ≈`#8a7355`
- 边框 `--line`：深木 ≈`#6b4a2b`（粗 3px 用）
- 强调 `--accent`：泰拉瑞亚金 ≈`#d4a017`；`--accent2` 亮金 ≈`#ffd23f`（标题/选中态/进度填充）
- 状态色 ok/wait/done 保持语义色相，调成像素风的高饱和平涂，两主题下校准对比度

### 配色 token——🌙 夜间「油灯木屋」
- 台面 `--desk`：近黑深木 ≈`#2a1d12`；面板 `--panel` ≈`#3a2c1d`、`--panel-2` ≈`#463624`
- 墨色 `--ink`：浅米 ≈`#f0dfc0`；强调转油灯橙 ≈`#ff9d3d` + 暖金
- 盘面相关变量（`--board-base`、`--peg-*`、`--cell-*`、`--bead-glow`）**两主题均保留现值**——盘面不参与像素化

具体色值实现时可微调，以「对比度可读 + 一眼木屋」为准。

### 像素造型手法（纯 CSS，零图片素材）
- **方角**：`border-radius` 全部清零；重要容器（面板/弹窗/槽位）用 `clip-path: polygon(...)` 切 4px 阶梯角
- **硬阴影**：柔阴影全部改 `4px 4px 0` 纯色无模糊偏移（小元素 2px）
- **木框 bevel**：3px 实线 `--line` + `box-shadow: inset` 上亮下暗各一道，造「上受光下背光」立体感
- **状态切换即时化**：去掉绝大多数 `transition`；hover＝金色描边亮起，active＝整体下沉 2px（硬阴影同步缩短）；脉搏等呼吸动画改 `steps()` 步进闪烁
- **进度条**：圆角平滑条 → 分段方块逐格点亮（`repeating-linear-gradient` 做格缝），金色填充
- **虚线框**（上传区）：加粗 `dashed` 像素点划感

### 字体
- `@font-face` 引入 Fusion Pixel 12px proportional zh_hans woff2，`font-display: swap`
- 回退链保留现有系统字体栈：字体加载中/失败时退回系统字，界面不开天窗
- 字号尽量取 12/24px 整倍数保锐利；非整倍字号实现时逐个过目，能归整的归整
- **emoji 功能图标（🌙❓💡🧽 等）保留不动**——替换为像素图标是独立工程，本期不做

## 二、组件级改动

1. **顶栏 header**：横向木板条（拼缝纹理）；标题金色像素字；右侧按钮为羊皮纸底+木框方块按钮。
2. **工具分段控件**（💡/🖌/🧽）：泰拉瑞亚物品快捷栏——一排紧贴方形槽位，选中格金色边框亮起。
3. **调色板（豆子盒）**：每个色号 = **库存物品槽**：方形槽位、木框 bevel、右下角像素数字显示剩余豆数（物品堆叠数语义）；槽内圆豆保持圆形（槽是像素的，物品是「真豆」）；选中槽金框高亮。
4. **底部操作台 dock**：木质工具栏；缩放滑杆改方块滑轨+方形把手；进度条按分段方块方案。
5. **CC 面板（右侧）**：木框面板 + 羊皮纸会话小卡；状态 pill 改方角像素徽章；活跃脉搏圆点改方点 `steps()` 闪烁。
6. **弹窗 sheet**（选图案/上传/帮助/快捷键）：大张羊皮纸 + 粗木框 + 切角，标题金色；遮罩保持半透明黑。
7. **作品廊**：挂画画框 → 像素木画框；钥匙扣挂环 → 像素挂环；作品图片本身不动。
8. **toast**：深木牌匾 + 浅米字，方角硬阴影。
9. **盘面边界**：盘面/洞洞板/豆子完全不动；其所在桌面换木纹；夜间豆子辉光照旧。

JS 侧零改动（canvas 经 `applyThemeVars()` 读 CSS 变量的管道原样）；唯一例外是 `server.js` 加字体静态路由。

## 三、基础设施

- 新增 `assets/fonts/fusion-pixel-12px-proportional-zh_hans.woff2`（来源：GitHub TakWolf/fusion-pixel-font releases）+ 同目录附 OFL-1.1 许可证文本（仓库 MIT 与 OFL 字体文件共存，许可证文本必须随文件入库）。
- `server.js` 新增 `/assets/` 静态路由：目录白名单、拒绝路径穿越（`..`）、正确 MIME（`font/woff2`）、`Cache-Control` 长缓存。
- README「文件结构」一节补 `assets/`，并注明字体来源与许可证。
- 本项目原则「单文件零依赖」就此放宽为：**运行时仍零 npm 依赖；新增一个静态字体资源文件**（字体缺失时优雅降级，见下）。

## 四、错误处理

- **字体文件缺失/离线拷贝丢失**：`font-display: swap` + 系统字体回退链，界面正常只是字体不像素；不报错不阻塞。
- **路由安全**：`/assets/` 之外路径不受影响；穿越尝试返回 404。

## 五、验证计划

- `node --test` 既有单测照跑（不涉 UI，应全绿）。
- 预览实例逐项截图核对，**日间 × 夜间各一遍**：主界面、选图案弹窗、上传弹窗、帮助浮层、作品廊（挂画+钥匙扣）、toast、CC 面板三态卡片（curl 向 `/hook` 喂假事件造出运行中/等输入/已完成）。
- 视觉回归重点：弹窗内滚动、调色板多色换行、CC 面板长列表、窄窗口布局。
- 字体路由：curl 验证 200 + `font/woff2` MIME；临时移走字体文件验证回退无报错。
